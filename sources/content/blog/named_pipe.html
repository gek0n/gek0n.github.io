--- title: Создание именованного пайпа в привилегированном процессе description: Статья описывает как привилегированный процесс должен задать права при создании именованного пайпа таким образом, чтобы непривилегированный процесс мог взаимодействовать с ним keywords: named pipe, pipe, name, windows, c, security, access level, privilege tags: - WIP date: 23.03.2018 --- Стандартный способ открытия пайпа через Winapi: ```c void main(uint32_t argc, char *argv[]) { return 0; } ``` При использовании этого способа пайп должен быть создан следующим образом: ```c void main(uint32_t argc, char *argv[]) { return 0; } ``` Если приложение, создавшее пайп является привилегированным, то операции с пайпом сможет выполнять только другое привилегированное приложение, приэтом непривилегированному приложению будет отказано в доступе. Чтобы позволить использовать пайп, созданный привилегированным процессом, в непривилегированном процессе, необходимо при создании пайпа задать аттрибуты безопасности: ```cpp void main(uint32_t argc, char *argv[]) { return 0; } ```
